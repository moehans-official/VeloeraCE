# Governance

本项目采用轻量社区治理模型。

## Maintainers

- Maintainer 负责：
  - PR 审核与合并
  - 版本发布
  - 安全问题响应
  - 路线图维护

## Decision Making

- 常规改动：Maintainer 审核通过即可合并。
- 重大改动（架构、兼容性破坏、许可证相关）：
  - 先开 Issue 或 Discussion
  - 给出可执行方案与迁移影响
  - 经 Maintainer 共识后执行

## Release Policy

- 使用语义化版本（SemVer）
- 建议节奏：
  - Patch：按需发布
  - Minor：功能稳定后发布
  - Major：仅在确有破坏性变更时发布

## Security

- 安全漏洞请不要公开提 Issue。
- 请通过私下渠道联系维护者，确认修复窗口后再披露。

## Inactive Maintainers

- 连续较长时间无响应的 Maintainer 可由社区提名替换。
- 交接应包含发布权限、仓库管理权限和文档说明。
